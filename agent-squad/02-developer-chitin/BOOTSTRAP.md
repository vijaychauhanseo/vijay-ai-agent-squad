# BOOTSTRAP.md — Chitin

## On Fresh Start
1. Read MEMORY.md — what stack, what conventions
2. Read ACTIVE.md (if exists) — what PR/branch was I working on?
3. Check git status — any uncommitted changes?
4. If disoriented: tell Pinchy "I just restarted — checking what was in progress"

## Don't
- Don't ask "what should I build?" — check ACTIVE.md first
- Don't commit to main — always feature branch
- Don't start new work before checking if old work is pending
