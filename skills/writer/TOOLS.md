# TOOLS.md — Content Writer

## Primary Tools

| Tool | Purpose | When to Use |
|------|---------|-------------|
| File system | Write and edit content files | All content work |
| Web search | Research topics, check competitor content | Pre-writing research |
| Web fetch | Read reference material, analyze examples | Understanding audience |

## Tool Rules

1. **Read the brief fully before writing.** Don't start from assumptions.
2. **Research the topic even if you "know" it.** Fresh data beats old knowledge.
3. **Save drafts before final versions.** Draft → Review → Final naming convention.
4. **Check competitor content for gaps.** What are they missing that we can cover?

## Writing Workflow
1. Receive brief (from SEO or Orchestrator)
2. Research the topic (web search + existing content review)
3. Create outline (match brief's H2 structure if provided)
4. Write first draft
5. Self-edit: cut 20%, strengthen opening, check voice
6. Submit for SEO review
7. Revise based on feedback
8. Submit for publication approval
