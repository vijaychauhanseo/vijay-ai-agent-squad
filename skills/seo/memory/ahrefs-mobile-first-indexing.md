## Core Concept
Google now uses the mobile version of your site as the primary version for indexing and ranking. If your mobile site has less content, slower speed, or missing elements vs. desktop — your rankings suffer.

## Key Takeaways
- Google is 100% mobile-first — the desktop version is now secondary
- Mobile and desktop versions must have identical content — hiding content on mobile hurts rankings
- Common failures: lazy-loaded images not indexed, content hidden behind tabs/accordions on mobile
- Page speed on mobile is a direct ranking factor (Core Web Vitals)
- Check if your site is mobile-first indexed: Google Search Console → Settings → Crawl Stats
- Use Google's Mobile-Friendly Test to identify issues

## Frameworks/Methods Covered
**Mobile-First Indexing Best Practices**:
1. Ensure desktop and mobile have identical content (don't hide sections on mobile)
2. Use responsive design (not separate m.domain.com)
3. Ensure lazy-loaded images are crawlable
4. Match structured data on both mobile and desktop
5. Verify all metadata is identical on both versions
6. Optimize Core Web Vitals for mobile: LCP < 2.5s, FID < 100ms, CLS < 0.1
